Sztuczna inteligencja (AI) odnosi się do systemów przeznaczonych do wykonywania zadań, które zwykle wymagają ludzkiej inteligencji — takich jak rozumowanie, rozwiązywanie problemów, postrzeganie i interpretacja języka.

Aplikacja sztucznej inteligencji to rozwiązanie programowe, które używa technik sztucznej inteligencji, takich jak przetwarzanie obrazów, mowa i wyodrębnianie informacji, do wykonywania zadań, które zwykle wymagają analizy podobnej do człowieka. Te aplikacje mogą zrozumieć, rozumować, uczyć się i reagować na dane wejściowe w sposób bardziej adaptacyjny niż tradycyjne oprogramowanie.

Aplikacje sztucznej inteligencji są obsługiwane przez modele uczenia maszynowego (ML), które są systemami matematycznymi wyszkolonymi w celu rozpoznawania wzorców w danych i przewidywania lub generowania danych wyjściowych. Modele uczenia maszynowego to silniki aplikacji sztucznej inteligencji. Podczas interakcji z aplikacją sztucznej inteligencji model wykonuje wnioskowanie, co oznacza, że stosuje informacje poznane podczas trenowania do nowych danych wejściowych.

Aplikacje sztucznej inteligencji to:

Oparte na modelu: używają wytrenowanych modeli do przetwarzania danych wejściowych i generowania danych wyjściowych, takich jak tekst, obrazy lub decyzje.
Dynamiczne: w przeciwieństwie do programów statycznych aplikacje sztucznej inteligencji mogą poprawić się w czasie poprzez ponowne trenowanie lub dostrajanie.
Oto kilka przykładów aplikacji sztucznej inteligencji dla różnych branż:

Opieka zdrowotna: narzędzia diagnostyczne oparte na sztucznej inteligencji, które analizują obrazy medyczne (takie jak zdjęcia rentgenowskie lub MRI) i pomagają lekarzom wykrywać choroby dokładniej i szybko.
Finanse: Systemy wykrywania oszustw, które używają sztucznej inteligencji do monitorowania transakcji w czasie rzeczywistym i identyfikowania podejrzanych działań, pomagając zapobiegać przestępstwom finansowym.
Sprzedaż detaliczna: Spersonalizowane systemy rekomendacyjne, które analizują zachowanie i preferencje klientów, aby sugerować produkty, zwiększając komfort zakupów.
Produkcja: Rozwiązania do konserwacji predykcyjnej, które używają sztucznej inteligencji do monitorowania sprzętu i prognozowania, kiedy maszyny mogą zakończyć się niepowodzeniem, co zmniejsza koszty przestojów i konserwacji.
Edukacja: Inteligentne systemy korepetycji, które dostosowują się do stylu i tempa nauki każdego ucznia, zapewniając dostosowane opinie i wsparcie w celu zwiększenia wyników uczenia się.
W tym module dowiesz się, jak firma Microsoft umożliwia tworzenie aplikacji sztucznej inteligencji przy użyciu najnowszej technologii, bezpiecznej i na dużą skalę. Chociaż model jest silnikiem, aplikacje sztucznej inteligencji wymagają również zabezpieczeń, sieci, hostingu, przechowywania danych, logiki aplikacji i interfejsów użytkownika. Firma Microsoft udostępnia całą infrastrukturę i usługi potrzebne do obsługi opracowywania sztucznej inteligencji w skali przedsiębiorstwa. Moduł zawiera podstawy dotyczące usprawniania tworzenia aplikacji sztucznej inteligencji na platformie Azure, integrowania ich z rozwiązaniem Microsoft Foundry i zapewnia szybkie innowacje.

Platforma Microsoft Azure jest jedną z wiodących na świecie platform w chmurze. Platforma w chmurze to kolekcja usług, których można używać za pośrednictwem Internetu, zamiast uruchamiać wszystko na własnym komputerze lub serwerze. Obecnie większość nowoczesnych aplikacji działa w chmurze, w internetowych centrach danych, które umożliwiają przechowywanie danych, uruchamianie kodu i skalowanie bez obaw o sprzęt fizyczny. Zamiast kupować i utrzymywać własną infrastrukturę, używasz usług udostępnianych przez zaufane platformy.

Platforma Azure jest zaufana przez organizacje na całym świecie do tworzenia bezpiecznych i niezawodnych aplikacji. Za pomocą platformy Azure można tworzyć aplikacje i agenci sztucznej inteligencji, którzy korzystają z zaawansowanych usług sztucznej inteligencji i wdrażać je na dużą skalę globalną.

Platforma Azure oferuje cztery podstawowe kategorie usług:

Przetwarzanie: zdolność do uruchamiania aplikacji, programów i obciążeń w chmurze. Obliczenia można traktować jako wynajem komputerów w chmurze, które można skalować w górę lub w dół, gdy tylko potrzebujesz.

Przechowywanie: Usługi, które umożliwiają zapisywanie i zarządzanie danymi w chmurze. Magazyn może zawierać pliki, bazy danych, obrazy, kopie zapasowe — wszystko, co ma być bezpiecznie przechowywane i dostępne z dowolnego miejsca.

Sieć: narzędzia łączące zasoby w chmurze ze sobą, z Internetem lub z organizacją.
Sieć zapewnia, że aplikacje mogą bezpiecznie i wydajnie komunikować się ze sobą.

App Services: gotowe platformy do tworzenia, hostowania i uruchamiania aplikacji bez zarządzania serwerami bazowymi.

Struktura organizacyjna platformy Azure
Gdy użytkownik zaczyna korzystać z platformy Azure, jego dostęp musi być zorganizowany i zarządzany. Platforma Azure organizuje dostęp i zarządzanie nimi według dzierżaw, subskrypcji, grup zasobów i zasobów.

Zrzut ekranu przedstawiający diagram struktury organizacyjnej platformy Azure.

Dzierżawa Azure jest podstawową bazą i tożsamością organizacji w chmurze Microsoftu. Posiadanie dzierżawy przypomina posiadanie jednostki mieszkalnej w dużym budynku, w którym budynek jest chmurą firmy Microsoft. Każdy najm jest oddzielony i zabezpieczony przed innymi — Twoja organizacja ma własne zamki, pokoje i mechanizmy kontrolne.

Gdy firma zarejestruje się w usłudze Azure lub Microsoft 365, Microsoft utworzy dla niej dzierżawcę. Ta dzierżawa to dedykowana i bezpieczna przestrzeń, w której znajdują się wszystkie zasoby chmurowe organizacji. Dzierżawca obejmuje użytkowników, grupy, tożsamości i zasady zarządzania bezpiecznym dostępem.

Subskrypcja platformy Azure to kontener rozliczeniowy dla zasobów w chmurze. Jedna dzierżawa może zawierać jedną lub wiele subskrypcji platformy Azure. Subskrypcja wiąże użycie platformy Azure z formą płatności (lub środki na bezpłatne/próbne konta) i ustawia granice kosztów, przydziałów i kontroli dostępu.

Grupy zasobów to foldery, które przechowują powiązane zasoby platformy Azure, dzięki czemu można nimi zarządzać razem. Jedna subskrypcja może mieć wiele grup zasobów. Każda grupa zasobów może mieć niestandardowe uprawnienia i zasady na poziomie grupy zasobów.

Zasób platformy Azure to dowolna pojedyncza usługa lub obiekt tworzony na platformie Azure, taki jak konto magazynu, baza danych lub zasób usługi Foundry. Każdy zasób ma typ zasobu, ustawienia konfiguracji, unikatową nazwę zasobu i identyfikator oraz opcjonalne mechanizmy kontroli dostępu. Typ zasobu (na przykład: Microsoft.Storage/storageAccounts) definiuje zachowanie, możliwości i ustawienia zasobu.

Podczas konfigurowania zasobu należy wybrać jego ustawienia, takie jak:

Region (w którym wdrożono zasób)
Warstwa wydajności (związana z kosztami)
Uprawnienia i bezpieczeństwo
Struktura organizacyjna platformy Azure pomaga zapewnić przejrzystość, bezpieczeństwo i skalowalność w środowiskach chmury. Dzierżawcy i subskrypcje umożliwiają wyraźne rozdzielenie kwestii między działami lub projektami. Grupy zasobów upraszczają zarządzanie przez grupowanie powiązanych zasobów, co ułatwia stosowanie zasad, monitorowanie użycia i automatyzowanie wdrożeń. Zrozumienie tej hierarchii jest niezbędne do wydajnego zapewniania ładu w chmurze i kontroli kosztów na platformie Azure.

Bezpieczeństwo i sieć
Zrzut ekranu przedstawiający ikony zabezpieczeń i sieci platformy Azure.

Podstawą aplikacji sztucznej inteligencji jest bezpieczeństwo i sieć. Platforma Azure jest bezpieczna zgodnie z projektem, oferując wbudowaną tożsamość, kontrolę dostępu i izolację sieci w celu ochrony rozwiązań sztucznej inteligencji.

Aplikacje sztucznej inteligencji wymagają silnej ochrony i stabilnej łączności. Narzędzia zabezpieczeń platformy Azure chronią aplikacje sztucznej inteligencji przed nieautoryzowanym dostępem i zagrożeniami. Na przykład Azure Entra ID gwarantuje, że wyłącznie uprawnione osoby i usługi mogą uzyskiwać dostęp do zasobów sztucznej inteligencji. Umożliwia ona kontrolę dostępu opartą na rolach (RBAC) w celu ograniczenia dostępu do wdrożeń modeli, zasobów i danych.

Platforma Azure chroni również tajne informacje aplikacji AI. Sekret tajny to każda wrażliwa wartość, którą aplikacja sztucznej inteligencji musi chronić, ponieważ udziela dostępu do systemu, usługi lub danych. Wpisy tajne mogą obejmować klucze interfejsu API, parametry połączenia bazy danych, tokeny OAuth, hasła i inne. Klucz to typ wpisu tajnego, który jest zwykle długim, losowo generowanym ciągiem — który uwierzytelnia żądanie podczas wywoływania punktu końcowego (adresu URL). Na platformie Azure wpisy tajne są zwykle przechowywane w usłudze Azure Key Vault, a nie w kodzie ani w usłudze GitHub.

Załóżmy na przykład, że tworzysz czatbota sztucznej inteligencji na platformie Azure.

Aplikacja wywołuje punkt końcowy modelu.
Twoje żądanie zawiera klucz do uwierzytelnienia wywołania.
Klucz jest przechowywany w usłudze Azure Key Vault jako wpis tajny.
Aplikacja pobiera sekret w trakcie działania z użyciem bezpiecznej metody (tożsamość zarządzana).
Narzędzia bezpieczeństwa platformy Azure obejmują ochronę tożsamości, tajemnic i danych, zgodność, wykrywanie zagrożeń, monitorowanie oraz kontrole bezpieczeństwa, takie jak zapory sieciowe. Jej usługi sieciowe zapewniają niezawodne, prywatne i wydajne działanie aplikacji w środowiskach chmurowych i hybrydowych.

Hosting i skalowanie
Zrzut ekranu przedstawiający ikony hostingu platformy Azure.

Aplikacje działają na komputerach lub środowiskach nazywanych hostem. W kontekście chmury host może być maszyną wirtualną (VM), zapewniającą moc obliczeniową, pamięć i sieć potrzebne do działania aplikacji.

Aby hostować i skalować aplikacje, możesz użyć usługi Azure Kubernetes Service (AKS) na potrzeby obciążeń konteneryzowanych lub usługi Azure App Service w celu szybkiego modernizacji i wdrażania aplikacji internetowych. Usługa AKS zarządza dużą liczbą kontenerów, które zawierają wszystko, czego potrzebuje kod do uruchomienia. Usługa Azure App Service hostuje aplikacje internetowe, interfejsy API i zadania w tle. Te usługi przyspieszają czas obrotu przy jednoczesnym zapewnieniu niezawodności.

Skalowanie aplikacji oznacza automatyczne lub ręczne dostosowanie ilości mocy obliczeniowej używanej przez aplikację — zwykle przez dodawanie lub usuwanie wystąpień. W aplikacjach w chmurze "instancje" odnoszą się do kopii twojej aplikacji działających jednocześnie.

Dwa typy skalowania:

Skalowanie poziome: dodawanie większej liczby instancji
Skalowanie w górę (pionowe): zwiększenie procesora/pamięci w istniejącym wystąpieniu
Platformy w chmurze, takie jak Platforma Azure, umożliwiają automatyczne skalowanie na podstawie użycia procesora CPU, liczby żądań lub metryk niestandardowych.

Microsoft Foundry to ujednolicona platforma jako usługa klasy korporacyjnej (PaaS) służąca do tworzenia, wdrażania i zarządzania aplikacjami i agentami sztucznej inteligencji. Konsoliduje modele, orkiestrację agentów, monitorowanie i narzędzia do zarządzania na jednej platformie, oferując infrastrukturę i zabezpieczenia na poziomie klasy produkcyjnej.

Platforma Foundry oferuje zaawansowane możliwości dla deweloperów, w tym możliwość wyboru spośród szerokiej gamy modeli, używanie tych modeli do tworzenia agentów, łączenie tych agentów z narzędziami i integrowanie wiedzy przy użyciu technologii IQ firmy Foundry, scentralizowanego punktu połączenia dla źródeł danych.

Zrzut ekranu przedstawiający elementy w narzędziu Foundry, w tym ikony modeli, narzędzi, agentów, narzędzi i wiedzy.

Models
Platforma Foundry obsługuje tysiące modeli — w tym rozbudowane opcje własne, innych firm oraz typu open source — bezpośrednio z poziomu ujednoliconego katalogu modeli. Deweloperzy mogą uzyskiwać dostęp do modeli OpenAI hostowanych na platformie Azure, takich jak najnowsza rodzina GPT-5 (GPT-5, GPT-5-mini, GPT-5-nano, GPT-5-chat/5.2-chat) z rozbudowanymi możliwościami wielomodalnymi i funkcjami rozumowania, wraz ze specjalistycznymi modelami z Anthropic (np. Claude Opus 4.6/4.5, Sonnet), Mistral, Cohere, Meta LLaMA, DeepSeek, xAI's Grok, Black Forest Labs i chronione (z ograniczonym dostępem, zarządzanie przedsiębiorstwem) modele Hugging Face.

Użytkownicy mogą przeglądać tysiące modeli — od ogromnych modeli podstawowych po lekkie, specyficzne dla domeny warianty — oceniać je za pomocą wbudowanych rankingów i placów zabaw oraz zarządzać wdrożeniami bezpośrednio w rozwiązaniu Foundry. Obsługa pełnego cyklu życia umożliwia wdrażanie w poszczególnych regionach, konfigurowalne typy wdrożeń (standardowe, aprowizowane, wsadowe), kontrolę wersji, i obsługę ładu dzięki odpowiedzialnej sztucznej inteligencji oraz bezpieczeństwu zawartości.

Agents
Podstawą rozwiązania Microsoft Foundry jest podejście oparte na agencie, które umożliwia deweloperom tworzenie inteligentnych, zorientowanych na zadania agentów bezpośrednio w projektach foundry. Ci agenci mogą przetwarzać dane wejściowe, wywoływać narzędzia, korzystać z danych i automatyzować przepływy pracy przy użyciu wbudowanej orkiestracji platformy. Platforma Foundry obsługuje podstawową koordynację — w tym wątkowanie wiadomości, wykonywanie narzędzi, kontrola bezpieczeństwa i monitorowanie — dzięki czemu deweloperzy mogą skupić się na projektowaniu celów i możliwości agenta. Korzystając z przepływów pracy z małą ilością kodu lub typu code-first, zespoły mogą tworzyć systemy wieloaplikowe, które współpracują z zasobami projektu, takimi jak dokumenty, zestawy danych, indeksy wyszukiwania i połączenia z systemami zewnętrznymi, w tym integracje, takie jak Azure Functions lub Microsoft Fabric.

Tools
Platforma Foundry oferuje kompleksowy zestaw usług platformy Azure, takich jak mowa, wizja, język, analiza dokumentów i inne. Te narzędzia foundry udostępniają funkcje sztucznej inteligencji, które mogą być wbudowane w aplikacje internetowe lub mobilne w sposób prosty do zaimplementowania. Istnieje kilkanaście różnych usług, które mogą być używane oddzielnie lub razem w celu dodania możliwości sztucznej inteligencji do aplikacji. Możesz na przykład użyć usługi Azure Vision do analizowania obrazów, języka platformy Azure w celu podsumowania tekstu, klasyfikowania informacji lub wyodrębniania kluczowych fraz oraz usługi Azure Speech w celu konwertowania mowy na tekst i tekst na mowę.

Wiedza
Foundry IQ zapewnia warstwę wiedzy obsługującą uprawnienia, opartą na wielu źródłach, która dostarcza agentom dokładne, oparte na danych odpowiedzi przy wykorzystaniu danych własnych organizacji. Umożliwia utworzenie konfigurowalnej bazy wiedzy zawierającej wewnętrzne i zewnętrzne źródła wiedzy, takie jak Azure Blob Storage, SharePoint, OneLake lub publiczne dane internetowe — i automatycznie obsługuje indeksowanie, fragmentowanie dokumentów, osadzanie wektorów i wyodrębnianie metadanych. Gdy agent wysyła zapytanie do bazy wiedzy, funkcja Foundry IQ wykorzystuje agentowe pobieranie do rozbicia pytania na podzapytania, przeszukania wielu źródeł równolegle oraz zwrócenia przygotowanych informacji z oparciem na cytatach, przy jednoczesnym respektowaniu uprawnień użytkowników oraz etykiet poufności Microsoft Purview. Dzięki temu agenci mogą czerpać z zaufanej, aktualnej zawartości i zwracać tylko informacje, które użytkownik ma uprawnienia do zobaczenia, zapewniając niezawodną podstawę wiedzy dla przepływów pracy sztucznej inteligencji przedsiębiorstwa.

Omówienie punktów końcowych
Podobnie jak w przypadku większości usług w chmurze, zasoby rozwiązania Microsoft Foundry są dostępne za pośrednictwem punktu końcowego interfejsu API reprezentującego punkt wejścia usługi. W tym module dowiesz się, jak używać sztucznej inteligencji do analizowania tekstu w dokumentach i innych zapisanych zasobach.

Przetwarzanie tekstu i analiza są możliwe za pośrednictwem przetwarzania języka naturalnego (NLP). NlP umożliwia maszynom zrozumienie, interpretowanie i reagowanie na język ludzki. Celem nlp jest analizowanie i wyodrębnianie znaczenia lub struktury z istniejącego tekstu.

Analiza tekstu to proces automatycznego badania tekstu pisanego w celu wyodrębnienia przydatnych informacji, takich jak tonacja, słowa kluczowe, jednostki lub tematy. Analiza tekstu opiera się na nlP, aby przekształcić tekst bez struktury w znaczące szczegółowe informacje.

Rozważ niektóre z tych aplikacji analizy tekstu:

Analiza opinii klientów: Firmy muszą analizować duże ilości opinii klientów, zgłoszeń do pomocy technicznej i odpowiedzi z ankiet. Dzięki technikom analizy tekstu mogą identyfikować trendy, wcześnie wykrywać niezadowolenie i ulepszać doświadczenia klientów.

Healthcare Text Analysis: Systemy opieki zdrowotnej muszą wyodrębnić informacje kliniczne z dokumentów medycznych bez struktury. Techniki analizy tekstu mogą pomóc zidentyfikować objawy, leki i diagnozy, wspieranie szybszego i dokładniejszego podejmowania decyzji.

Przetwarzanie dokumentów finansowych: Banki i instytucje finansowe obsługują duże ilości umów, wniosków o pożyczki i dokumentów regulacyjnych. Techniki analizy tekstu mogą pomóc w automatycznym wyodrębnianiu kluczowych szczegółów, takich jak stopy procentowe, informacje o kredytobiorcy i ryzyko związane z zgodnością , co pomaga skrócić czas ręcznego przeglądu i poprawić dokładność.

Podsumowanie dokumentów prawnych: Kancelarie prawnicze zarządzają długimi aktami spraw, orzeczeniami sądowymi i umowami. Techniki analizy tekstu mogą pomóc w podsumowywać złożone teksty prawne, wyróżniać ważne klauzule i klasyfikować dokumenty według tematu, umożliwiając adwokatom wydajniejsze pracę i szybsze podejmowanie świadomych decyzji.

# Load environment variables from .env file

deployment_name = os.getenv("MODEL_DEPLOYMENT_NAME")

# Create the client object

# Print the results

Zrozumienie MCP
Protokół MCP (Model Context Protocol) to otwarty standard definiujący sposób łączenia agentów sztucznej inteligencji z zewnętrznymi narzędziami i źródłami danych. Pomyśl o MCP jak o uniwersalnej karcie: zamiast pisać niestandardowy kod integracji dla każdej usługi, której potrzebuje agent, należy połączyć agenta z serwerem MCP, który już uwidacznia te możliwości w standardowy sposób.

McP używa architektury klient-serwer:

Klient MCP jest agentem sztucznej inteligencji (lub aplikacją hosta, na którym działa agent). Wysyła żądania i odbiera odpowiedzi.
Serwer MCP to usługa, która uwidacznia narzędzia, dane lub akcje. Nasłuchuje żądań, wykonuje odpowiednią funkcję i zwraca ustrukturyzowany wynik.
Gdy agent łączy się z serwerem MCP, może wykryć, jakie narzędzia oferuje serwer i wywołać je w razie potrzeby — bez pracy z niestandardową integracją. Serwer może odpowiedzieć na żądanie przez:

Dostarczanie danych (na przykład: wyniki analizy sentymentu, kluczowe frazy lub rekordy jednostek)
Podejmowanie akcji (na przykład: przetwarzanie partii dokumentów)
Ta separacja problemów zapewnia czystą logikę agenta i ułatwia zamianę lub rozszerzanie możliwości dzięki połączeniu z różnymi serwerami MCP.

Serwer MCP języka Azure
Serwer MCP języka Azure jest usługą zarządzaną, która uwidacznia możliwości Azure Language in Foundry Tools poprzez MCP. Działa on jako most między agentem a pełnym zestawem funkcji językowych Azure — rozpoznawanie nazwanych jednostek, analiza tonacji, wykrywanie języka i nie tylko.

Ponieważ serwer jest zgodny ze standardem MCP, agent może wywołać te narzędzia do analizy języka przy użyciu tego samego protokołu, który jest używany dla dowolnego innego serwera MCP. Nie musisz wywoływać interfejsu API REST języka Azure bezpośrednio ani zarządzać tokenami uwierzytelniania w kodzie agenta.

Python
import azure.cognitiveservices.speech as speechsdk

# Set up the speech config using resource endpoint
endpoint_url = "ENDPOINT"
speech_key = "FOUNDRY_KEY"

speech_config = speechsdk.SpeechConfig(
    subscription=speech_key,
    endpoint=endpoint_url
)

# Create a recognizer with microphone input
audio_config = speechsdk.audio.AudioConfig(use_default_microphone=True)
speech_recognizer = speechsdk.SpeechRecognizer(
    speech_config=speech_config, 
    audio_config=audio_config
)

# Event handlers
def recognized_handler(evt):
    print(f"Recognized: {evt.result.text}")

def recognizing_handler(evt):
    print(f"Recognizing: {evt.result.text}")

# Connect event handlers
speech_recognizer.recognized.connect(recognized_handler)
speech_recognizer.recognizing.connect(recognizing_handler)

# Start continuous recognition
speech_recognizer.start_continuous_recognition()
print("Say something...")

# Keep the program running
input("Press Enter to stop...")
speech_recognizer.stop_continuous_recognition()

Transkrypcja w czasie rzeczywistym: zamiana mowy w czasie rzeczywistym na tekst umożliwia transkrypcję strumieni audio do tekstu. Transkrypcję w czasie rzeczywistym można wykorzystać do prezentacji, pokazów lub innych sytuacji, w których ktoś mówi.

Aby transkrypcja w czasie rzeczywistym działała, aplikacja musi nasłuchiwać przychodzącego dźwięku z mikrofonu lub innego źródła danych wejściowych audio, takich jak plik audio. Kod aplikacji przesyła strumieniowo dźwięk do usługi, co zwraca transkrypowany tekst.

Transkrypcja wsadowa: nie wszystkie scenariusze zamiany mowy na tekst są w czasie rzeczywistym. Nagrania audio mogą być przechowywane w udziale plików, serwerze zdalnym, a nawet w usłudze Azure Storage. Możesz wskazać pliki audio z identyfikatorem URI sygnatury dostępu współdzielonego (SAS) i asynchronicznie odbierać wyniki transkrypcji.

Transkrypcja wsadowa powinna być uruchamiana w sposób asynchroniczny, ponieważ zadania wsadowe są zaplanowane na podstawie najlepszego nakładu pracy. Zwykle zadanie rozpoczyna wykonywanie w ciągu kilku minut od żądania, ale nie ma oszacowania, kiedy zadanie zmienia się w stan uruchomienia.

import os
import azure.cognitiveservices.speech as speechsdk

# This example requires environment variables named "FOUNDRY_KEY" and "ENDPOINT"
speech_config = speechsdk.SpeechConfig(subscription=os.environ.get('FOUNDRY_KEY'), endpoint=os.environ.get('ENDPOINT'))
audio_config = speechsdk.audio.AudioOutputConfig(use_default_speaker=True)

# The neural multilingual voice can speak different languages based on the input text.
speech_config.speech_synthesis_voice_name='en-US-Ava:DragonHDLatestNeural'

speech_synthesizer = speechsdk.SpeechSynthesizer(speech_config=speech_config, audio_config=audio_config)

# Get text from the console and synthesize to the default speaker.
print("Enter some text that you want to speak >")
text = input()

speech_synthesis_result = speech_synthesizer.speak_text_async(text).get()

if speech_synthesis_result.reason == speechsdk.ResultReason.SynthesizingAudioCompleted:
    print("Speech synthesized for text [{}]".format(text))
elif speech_synthesis_result.reason == speechsdk.ResultReason.Canceled:
    cancellation_details = speech_synthesis_result.cancellation_details
    print("Speech synthesis canceled: {}".format(cancellation_details.reason))
    if cancellation_details.reason == speechsdk.CancellationReason.Error:
        if cancellation_details.error_details:
            print("Error details: {}".format(cancellation_details.error_details))
            print("Did you set the speech resource key and endpoint values?")

W edytorze kodu możemy utworzyć jeden plik języka Python, który zawiera kod aplikacji. Istotne jest, że potrzebujesz klucza zasobu Foundry, punktu końcowego oraz nazwy wdrożonego modelu.

 Notatka

Podczas wdrażania modelu w Foundry, model ma nazwę podstawową lub oryginalną oraz nadaną przez ciebie oryginalną nazwę wdrożenia. Foundry gospodaruje wdrożonym modelem (na przykład modelami klasy GPT z funkcją wizji) i udostępnia punkt końcowy.

W przykładzie kodu tworzysz klienta, wskazujesz jego punkt końcowy i przekazujesz nazwę wdrożenia modelu (nazwę, którą nadałeś modelowi) jako .

Python
import os
from openai import OpenAI

# Environment variables you set locally or in your app service:
FOUNDRY_KEY = "... your key ..."
ENDPOINT = "https://YOUR-RESOURCE-NAME.openai.azure.com/openai/v1/"
MODEL_NAME = "your-model-deployment-name"  # e.g., "gpt-4.1-mini" deployed as "my-vision-deploy"

client = OpenAI(
    api_key=os.getenv("FOUNDRY_KEY"),
    base_url=os.getenv("ENDPOINT"),
)

image_url = ""

response = client.responses.create(
    model=os.getenv("MODEL_NAME"),  # your deployment name 
    input=[
        {
            "role": "user",
            "content": [
                {"type": "input_text", "text": "What is in this image? Provide 3 bullet points."},
                {"type": "input_image", "image_url": image_url}
            ],
        }
    ],
)

print(response.output_text)

Możliwość dynamicznego generowania oryginalnych obrazów na podstawie opisów może być niezwykle cenna w scenariuszach obejmujących multimedia, publikowanie i tworzenie zawartości.

Aby wygenerować obrazy przy użyciu zestawu SDK języka Python openAI, potrzebne są następujące elementy:

Zasób odlewni
Wdrożony model z obsługą przetwarzania obrazów (nazwa wdrożenia jest przekazywana jako MODEL_NAME)
Uwierzytelnianie za pomocą klucza API lub Microsoft Entra ID
Wywołania API Odpowiedzi OpenAI zawierające obraz jako dane wejściowe (adres URL lub dane w formacie base64)
 Notatka

Base64 odnosi się do plików, takich jak obrazy, które są w postaci binarnej (nieprzetworzone bajty). Pliki JSON i adresy URL są tylko tekstem. Kodowanie Base64 konwertuje dane binarne na bezpieczny tekst ASCII, umożliwia osadzanie plików binarnych w formacie JSON lub adresach URL.

Rozważmy na przykład następujący kod w języku Python:

Python
import os
import base64
from openai import OpenAI

# Required environment variables (example names)
FOUNDRY_KEY="..."
ENDPOINT="https://YOUR-RESOURCE-NAME.openai.azure.com/openai/v1/"
MODEL_NAME="your-gpt-image-deployment-name"  # e.g., "gpt-image-1"

client = OpenAI(
    api_key=os.environ["FOUNDRY_KEY"],
    base_url=os.environ["ENDPOINT"],
)

prompt = "A modern flat illustration of a robot holding a potted plant, clean vector style, pastel colors."

response = client.responses.create(
    model=os.environ["MODEL_NAME"],  # your deployment name in Foundry
    input=prompt,
    tools=[{"type": "image_generation"}],
)

image_base64 = next(
    item.result for item in response.output
    if item.type == "image_generation_call"
)

with open("foundry_generated.png", "wb") as f:
    f.write(base64.b64decode(image_base64))

print("Saved: foundry_generated.png")

Usługa Microsoft Azure Content Understanding w narzędziu Foundry Tools używa sztucznej inteligencji do wyodrębniania informacji ustrukturyzowanych z zawartości bez struktury. Usługa Azure Content Understanding pomaga aplikacjom zrozumieć zawartość , identyfikując jednostki, pola, relacje i znaczenie zawartości.

Wideo: na przykład wideo dotyczące spotkań lub innych plików multimedialnych
Wyodrębnianie informacji opartych na sztucznej inteligencji w usłudze Azure Content Understanding automatyzuje proces przekształcania zawartości bez struktury na dane czytelne dla maszyny, które można przeszukiwać i analizować. Jak działa usługa Azure Content Understanding
Usługa Azure Zrozumienie Treści wykorzystuje modelowy przepływ pracy do wyodrębniania danych, w którym zawartość bez struktury jest pozyskiwana, analizowana i zwracana jako dane ustrukturyzowane.

Pozyskiwanie zawartości: przesyłasz zawartość do usługi Azure Content Understanding.

Analiza oparta na sztucznej inteligencji: usługa używa kombinacji: optycznego rozpoznawania znaków (OCR), rozpoznawania mowy, interpretacji języka naturalnego i wielomodalnych modeli sztucznej inteligencji do analizowania zawartości.

Dane wyjściowe ze strukturą: usługa zwraca wyniki ustrukturyzowane (na przykład w formacie JSON), które pasują do modelu — co ułatwia przechowywanie, wyszukiwanie lub integrowanie danych z systemami podrzędnymi.

 Notatka

JSON (JavaScript Object Notation) to tekstowy format danych używany do przechowywania i wymiany danych ustrukturyzowanych między systemami. Łatwo jest ludziom czytać i pisać, a maszynom łatwo analizować i generować.

Omówienie schematów
Optyczne rozpoznawanie znaków (optyczne rozpoznawanie znaków) umożliwia komputerowi odczytywanie tekstu ze zdjęć, takich jak zeskanowane dokumenty, zdjęcia paragonów lub obrazy stron drukowanych, a następnie przekształcenie tego tekstu w tekst cyfrowy z możliwością edycji i przeszukiwania. Podstawowa funkcja OCR ułatwia rozpoznawanie tekstu drukowanego, skupia się na wyodrębnianie tekstu i nie rozumie znaczenia, kontekstu ani relacji między wyrazami.

Możliwości analizy dokumentów usługi Azure Content Understanding wykraczają poza proste wyodrębnianie tekstu opartego na protokole OCR w celu uwzględnienia wyodrębniania pól i ich wartości na podstawie schematu . Podejście oparte na schemacie odróżnia usługę Azure Content Understanding od podstawowych usług OCR lub transkrypcji.

Schemat opisuje informacje, które chcesz wyodrębnić i jak te informacje powinny być ustrukturyzowane. Podczas definiowania schematu należy określić pola do wyodrębnienia. Schemat zawiera listę określonych pól lub jednostek, które cię interesują.

Zrozumienie analizatorów
Analizator to jednostka w usłudze Azure Content Understanding, która pobiera dane wejściowe, stosuje analizę sztucznej inteligencji i generuje wyniki strukturalne. Analizatory generują również przewidywalne wyniki JSON. Wyniki ustrukturyzowane ułatwiają przetwarzanie podrzędne (magazyn, wyszukiwanie, automatyzacja).

Tworzenie aplikacji klienckiej za pomocą usługi Azure Content Understanding
Interfejs API usługi Content Understanding umożliwia utworzenie uproszczonej aplikacji klienckiej, która programowo wyodrębnia dane.

 Notatka

Aplikacja kliencka to program, który działa na urządzeniu użytkownika i żąda usług lub danych z innego systemu, zazwyczaj serwera za pośrednictwem sieci. Klient jest częścią aplikacji, z którą użytkownicy korzystają, podczas gdy serwer wykonuje ciężką pracę w tle. Aplikacje mogą żądać danych lub akcji z usługi i odbierać ustrukturyzowaną odpowiedź przy użyciu interfejsu API.

Korzystając z interfejsu API usługi Content Understanding, możesz wybrać wstępnie utworzony analizator lub utworzyć analizator niestandardowy. Wstępnie utworzone analizatory obejmują: prebuilt-invoice, prebuilt-imageSearch, prebuilt-audioSearch i prebuilt-videoSearch. Podczas przesyłania zawartości do analizy do analizatora analiza jest asynchroniczna, co oznacza, że wynik otrzymasz później, gdy będzie gotowy. Ponieważ analiza jest asynchroniczna, należy sondować adres URL Operation-Location (lub analyzerResults), dopóki zadanie nie powiedzie się.

Korzystanie z zestawu SDK języka Python usługi Azure Content Understanding
Przyjrzyjmy się procesowi używania zestawu SDK języka Python do analizowania faktury z adresu URL.

Zainstaluj zestaw SDK usługi Azure Content Understanding dla języka Python.
Bash
python -m pip install azure-ai-contentunderstanding
Zidentyfikuj punkt końcowy zasobu Foundry i klucz API lub Microsoft Entra ID. Punkt końcowy zwykle wygląda następująco: https://<your-resource-name>.services.ai.azure.com/

Utwórz i uruchom kod aplikacji klienckiej. Element analzyer_id jest identyfikatorem wstępnie utworzonego analizatora. Listę wstępnie utworzonych wartości identyfikatorów analizatora można znaleźć tutaj.

Python
import os
from azure.ai.contentunderstanding import ContentUnderstandingClient
from azure.core.credentials import AzureKeyCredential

endpoint = os.environ["FOUNDRY_ENDPOINT"]
key = os.environ["FOUNDRY_KEY"]

client = ContentUnderstandingClient(endpoint=endpoint, credential=AzureKeyCredential(key))

# 1) start analysis with analyzer id + inputs
analyzer_id = "prebuilt-invoice"
inputs = [
    {"url": "https://github.com/Azure-Samples/azure-ai-content-understanding-python/raw/refs/heads/main/data/invoice.pdf"}
]

# 2) wait for the Long Running Operation (LRO) to complete
poller = client.begin_analyze(analyzer_id=analyzer_id, inputs=inputs)  # starts LRO
result = poller.result()  # waits for completion (polling handled by SDK)

# 3) read structured fields + markdown
# The result typically includes extracted "fields" and "markdown" per input content item.
for content in result.contents:
    print(content.markdown)
    print(content.fields)

Przyjrzyjmy się przykładowi przy użyciu zestawu SDK języka Python. Po uruchomieniu następującego kodu analizuje plik audio przy użyciu wstępnie utworzonego analizatora. Wstępnie utworzony analizator jest identyfikowany jako prebuilt-audioSearch.

Python
import os
from azure.ai.contentunderstanding import ContentUnderstandingClient
from azure.core.credentials import AzureKeyCredential

# Endpoint and key for your Foundry resource
endpoint = os.environ["FOUNDRY_ENDPOINT"]  # e.g., "https://<resource>.services.ai.azure.com/"
key = os.environ["FOUNDRY_KEY"]

client = ContentUnderstandingClient(
    endpoint=endpoint,
    credential=AzureKeyCredential(key)
)

# Choose a prebuilt analyzer for audio
# (The documents module lists examples like prebuilt-audioSearch / prebuilt-videoSearch.)
analyzer_id = "prebuilt-audioSearch"

# Provide an input audio file (URL shown here; you can swap in your own accessible media URL)
inputs = [
    {"url": "https://<your-host>/samples/voicemail.wav"}
]

# Start analysis (asynchronous long-running operation)
poller = client.begin_analyze(analyzer_id=analyzer_id, inputs=inputs)

# Wait for completion (SDK polls under the hood)
result = poller.result()

# Inspect the structured output (JSON-like objects)
for content in result.contents:
    # Some analyzers may return a transcript and/or extracted fields depending on the analyzer and schema
    print("=== MARKDOWN / TRANSCRIPT (if provided) ===")
    print(getattr(content, "markdown", None))

    print("\n=== EXTRACTED FIELDS ===")
    print(getattr(content, "fields", None))
